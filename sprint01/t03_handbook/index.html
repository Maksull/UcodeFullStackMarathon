PROTOCOLS

  • HTTP methods

    GET - Retrieve a resource's representation/information.

    POST - Create a new resource into the collection of resources.

    PUT - Update an existing resource, or create a new one if it doesn't exist.

    DELETE - Delete resources identified by Request-URI.

    PATCH - Make partial updates on a resource.
  • HTTP status code

    1xx: Informational responses.
    Example: 100 Continue.

    2xx: Success responses.
    Example: 201 Created.

    3xx: Redirection responses.
    Example: 301 Moved Permanently.

    4xx: Client error responses.
    Example: 404 Not Found.

    5xx: Server error responses.
    Example: 500 Internal Server Error.
  • DNS main by types

    Domain Name System (DNS) resolves human-readable domain names to machine-readable IP addresses. It consists of multiple types of records, including:

    A (Address) Records - Maps a domain name to an IPv4 address.
    AAAA (IPv6 Address) Records - Maps a domain name to an IPv6 address.
    CNAME (Canonical Name) Records - Alias of one domain name to another.
    MX (Mail Exchange) Records - Specifies the mail server responsible for receiving email on behalf of a domain.
    TXT (Text) Records - Used to associate arbitrary text with a domain.
    PTR (Pointer) Records - Used in reverse DNS lookups to map an IP address to a domain name.

  • Identifying google's IP

    To identify Google's IP address:

    1. Open your terminal or command prompt.

    2. Type the command 'nslookup google.com' and press Enter.

    You'll receive the name and address of Google's servers.
    Name: google.com
    Address: 216.58.215.99
    Repeat the process with 'google.com.ua' for specific country-based results.
